# Output Types

## General Rule

Generate only the output type explicitly requested by the user.

Do not silently generate:

- both formal review comments and chapter annotations
- both Chinese and English versions

unless the user explicitly asks for both.

## 1. Formal Review Comments

### Recommended Chinese structure

- 标题
- 评阅对象
- 评阅身份
- 一、总体结论
- 二、上次反馈落实情况评估
- 三、主要审阅意见与修改建议
- 四、关于是否满足毕业条件的判断
- 五、送审前优先修改清单
- 六、评阅依据

### Recommended English structure

- Title
- Review target
- Review perspective
- I. Overall conclusion
- II. Assessment of whether prior issues were addressed
- III. Main review comments and revision suggestions
- IV. Judgment on whether the thesis appears to meet graduation expectations
- V. Priority revision list
- VI. Basis for the judgment

### When to use

Use this format when the user wants:

- a formal review opinion
- a supervisor-style review
- a complete evaluative statement for submission or revision

## 2. Chapter-By-Chapter Annotations

### Recommended Chinese structure

- 标题
- 说明
- 摘要与前置部分
- 第一章
- 第二章
- 第三章
- 第四章
- 第五章
- 第六章
- 参考文献与附录
- 总结性修改建议

### Recommended English structure

- Title
- Scope note
- Abstract and front matter
- Chapter 1
- Chapter 2
- Chapter 3
- Chapter 4
- Chapter 5
- Chapter 6
- References and appendix
- Final revision priorities

### Content style

Each chapter annotation should localize issues as specifically as the available text allows, for example:

- chapter
- section
- table
- figure
- opening sentence
- closing paragraph

Where helpful, provide:

- why the passage is weak
- what should change
- one suggested rewrite

## Language guidance

### Chinese output

Prefer formal academic Chinese with explicit evaluative labels.

### English output

Prefer graduate-thesis review language rather than casual editing language.

Avoid exaggerated claims such as:

- definitely publishable
- fully meets all requirements
- unquestionably ready

unless the evidence actually supports that level of certainty.
