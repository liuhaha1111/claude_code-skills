# Revision Comparison

## Use Case

Use this reference when the user provides:

- prior formal review comments
- prior chapter annotations
- earlier review drafts

and asks whether the revised thesis has properly addressed those issues.

## Comparison Method

### 1. Extract prior issues

Reduce the earlier review set into a clean issue list.

Each issue should state:

- where the original problem was
- what the problem was
- what revision was expected

### 2. Inspect the revised thesis

For each prior issue, find the matching part of the revised manuscript and classify it as:

- clearly corrected
- partially corrected
- not properly corrected

### 3. Use evidence-based phrasing

#### Clearly corrected

Use when the earlier problem is no longer present or has been directly and adequately fixed.

#### Partially corrected

Use when the author addressed the issue but:

- the change is incomplete
- the wording improved but the logic did not
- a formatting error was fixed but the substantive issue remains

#### Not properly corrected

Use when:

- the issue is still present
- the revision is superficial
- the new wording still fails to solve the core problem

## Output Guidance

In formal review comments, summarize the revision status under a dedicated section.

In chapter annotations, carry the old issue forward into the relevant chapter section and explain what still remains to be fixed.

## Common Pitfalls

- assuming an issue is fixed because terminology changed
- assuming an issue is unresolved without checking the revised chapter carefully
- judging “partially corrected” when the issue is actually unchanged
- failing to separate formatting fixes from substantive fixes
